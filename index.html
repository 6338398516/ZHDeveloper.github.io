链式编程思想

链式编程思想：把一系列的操作使用.来来连接成一句代码，可读性好。典型的代表Masonry框架，例如：

[redView mas_makeConstraints:^(MASConstraintMaker *make) {
    make.left.top.equalTo(@10);
    make.right.bottom.equalTo(@(-10));
}];

ZHCalculateMaker的声明与实现

#import <Foundation/Foundation.h>

@interface ZHCalculateMaker : NSObject

@property (nonatomic, assign) float result;

- (ZHCalculateMaker * (^)(float))add;

- (ZHCalculateMaker * (^)(float))subtract;

- (ZHCalculateMaker * (^)(float))multiply;

- (ZHCalculateMaker * (^)(float))divide;

@end

#import "ZHCalculateMaker.h"

@implementation ZHCalculateMaker

- (ZHCalculateMaker * (^)(float))add {
    return ^(float num) {
        self.result += num;
        return self;
    };
}

- (ZHCalculateMaker * (^)(float))subtract {
    return ^(float num) {
        self.result -= num;
        return self;
    };
}

- (ZHCalculateMaker * (^)(float))multiply {
    return ^(float num) {
        self.result *= num;
        return self;
    };
}

- (ZHCalculateMaker * (^)(float))divide {
    return ^(float num) {
        self.result /= num;
        return self;
    };
}

@end

NSObject+ZHCalculate.h的声明与实现

#import <Foundation/Foundation.h>
#import "ZHCalculateMaker.h"

@interface NSObject (ZHCalculate)

+ (float)makeCalculate:(void (^)(ZHCalculateMaker *maker))block;

@end

#import "NSObject+ZHCalculate.h"
#import "ZHCalculateMaker.h"

@implementation NSObject (ZHCalculate)

+ (float)makeCalculate:(void (^)(ZHCalculateMaker *maker))block {
    //初始化制造者
    ZHCalculateMaker *maker = [ZHCalculateMaker new];

    block(maker);

    return maker.result;
}

@end

    [super viewDidLoad];

        maker.add(3).multiply(5);

    }];

    NSLog(@"%f", result);
}

KVO底层实现

观察者模式（KVO），当对象的属性值发生变化，会通过调用方法observeValueForKeyPath:ofObject:change:context:来说明属性的值发生了改变。KVO的实现代码如下：

- (void)viewDidLoad {
[super viewDidLoad];

Person *p = [Person new];

p.name = @"张三";

[p addObserver:self forKeyPath:@"name" options:NSKeyValueObservingOptionNew context:nil];

    p.name = @"李思";
}

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSString *, id> *)change context:(void *)context {
    NSLog(@"%@", change);
}

从下图可以看出，当还未给对象添加观察者前，p对象是Person类。

当给对象添加观察者后，P对象的类变成为NSKVONotifying_Person。

注：只用调用属性的set方法来改变属性的值时，才会观察者才会知道属性值发生改变。

KVO底层实现的分析：

1.当为对象添加观察者时，动态创建NSKVONotifying_Person，NSKVONotifying_Person是Person类的子类。

2.修改当前对象的isa指针->NSKVONotifying_Person

3.当对象的属性值发生改变时，会调用NSKVONotifying_Person的set方法

4.重写NSKVONotifying_Person的set方法，调用观察者的方法。

重写KVO的实现：

NSObject (KVO)的声明与实现

//NSObject (KVO).h
#import <Foundation/Foundation.h>

@interface NSObject (KVO)

- (void)ZHAddObserver:(NSObject *)observer forKeyPath:(NSString *)keyPath options:(NSKeyValueObservingOptions)options context:(nullable void *)context;

@end

//NSObject (KVO).m
#import "NSObject+KVO.h"
#import <objc/runtime.h>
#import "ZHNSKVONotifying_Person.h"

@implementation NSObject (KVO)

- (void)ZHAddObserver:(NSObject *)observer forKeyPath:(NSString *)keyPath options:(NSKeyValueObservingOptions)options context:(nullable void *)context {
    //修改当前对象的isa指针
    object_setClass(self, [ZHNSKVONotifying_Person class]);

    //保存把观察者
    objc_setAssociatedObject(self, @"observer", observer, OBJC_ASSOCIATION_RETAIN_NONATOMIC);
}

@end

ZHNSKVONotifying_Person的声明与实现

#import "Person.h"

@interface ZHNSKVONotifying_Person : Person

@end

#import "ZHNSKVONotifying_Person.h"
#import <objc/runtime.h>

@implementation ZHNSKVONotifying_Person

- (void)setName:(NSString *)name {
    [super setName:name];

    id observer = objc_getAssociatedObject(self, @"observer");

    NSDictionary *change = @{ @"change" : name };

    [observer observeValueForKeyPath:@"name" ofObject:self change:change context:nil];
}

@end

实现调用

- (void)viewDidLoad {
    [super viewDidLoad];

    Person *p = [Person new];

    p.name = @"张三";

    [p ZHAddObserver:self forKeyPath:@"name" options:NSKeyValueObservingOptionNew context:nil];

    p.name = @"李思";
}

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSString *, id> *)change context:(void *)context {
    NSLog(@"%@", change);
}

iOS开发技巧

###Xcode IB小技巧

1.可视化坐标距离。我们使用StroyBoard或者Xib来布局界面时，当需要计算两个视图的距离，分别记下两个视图的坐标值，然后手动计算视图间的距离。这种方法确实能算出两个视图之间的距离，但明显这种方法不够优雅。Xcode的IB提供可视化距离的显示，鼠标单击选中第一个试图,按住Option键，鼠标移动到第二个试图上，便可以显示第一个视图相对于另一个试图的距离。

2.显示view的层次结构。如果我们想要查看一个试图的层次结构时，可以在左侧的层次结构表中查看。当一个视图的层次结构比较复杂时，则需要在左侧的层级菜单一级一级的点击进去。当然我们可以使用快捷键来显示一个view在当前的Xib中的层次结构。快捷键为control+shift。

3.添加辅助线。对于做设计的同学来学，辅助线是一个必不可少的利器。选中某个视图，同时按住cmd+shift+|，就会在选中的中间添加一条辅助线。当把鼠标放在显示的辅助线往左或者右拖拽就可以添加更多的辅助线，当不需要辅助线时，把辅助线往外拖就会把辅助线给销毁。

iOS屏幕旋转

当屏幕发生旋转，需要注册通知UIDeviceOrientationDidChangeNotification来告诉应用程序屏幕方向发生改变。UIDevice的beginGeneratingDeviceOrientationNotifications方法是告诉应用程序每次设备更改的方向。

//监听屏幕方向变化
- (void)listeningRotating {
    //当屏幕方向发生改变时通知应用程序
    [[UIDevice currentDevice] beginGeneratingDeviceOrientationNotifications];

    [[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(onDeviceOrientationChange) name:UIDeviceOrientationDidChangeNotification object:nil];
}

- (void)removeListeningRotating {
    [[NSNotificationCenter defaultCenter] removeObserver:self name:UIDeviceOrientationDidChangeNotification object:nil];
}

- (void)onDeviceOrientationChange {
    UIDeviceOrientation orientation             = [UIDevice currentDevice].orientation;
    UIInterfaceOrientation interfaceOrientation = (UIInterfaceOrientation)orientation;
    switch (interfaceOrientation)
    {
        case UIInterfaceOrientationPortraitUpsideDown:
        {
            //NSLog(@"第3个旋转方向---电池栏在下");
        }
        break;
        case UIInterfaceOrientationPortrait:
        {
            //NSLog(@"第0个旋转方向---电池栏在上");
        }
        break;
        case UIInterfaceOrientationLandscapeLeft:
        {
            //NSLog(@"第2个旋转方向---电池栏在右");
        }
        break;
        case UIInterfaceOrientationLandscapeRight:
        {
            //NSLog(@"第1个旋转方向---电池栏在左");
        }
        break;

        default:
            break;
    }
}

//强制横屏
- (void)interfaceOrientation:(UIInterfaceOrientation)orientation {
    // arc下
    if ([[UIDevice currentDevice] respondsToSelector:@selector(setOrientation:)])
    {
        SEL selector             = NSSelectorFromString(@"setOrientation:");
        NSInvocation *invocation = [NSInvocation invocationWithMethodSignature:[UIDevice instanceMethodSignatureForSelector:selector]];
        [invocation setSelector:selector];
        [invocation setTarget:[UIDevice currentDevice]];
        int val = orientation;
        [invocation setArgument:&val atIndex:2];
        [invocation invoke];

iOS8以后，从竖屏切换到横屏时，状态栏的默认是隐藏的。需要通过代码来设置显示手机的状态栏。

情况一：当状态的控制由UIController来管理时，可以通过重写控制器的prefersStatusBarHidden，来控制状态栏的显示或者隐藏。

- (BOOL)prefersStatusBarHidden {
    return NO;
}

情况二：从info.plist中设置View controller-based status bar appearance为NO，表示不由控制器来管理状态栏。

[[UIApplication sharedApplication] setStatusBarHidden:NO withAnimation:UIStatusBarAnimationNone];

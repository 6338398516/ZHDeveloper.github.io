subscripts

subscripts(下标): 访问对象中数据的快捷方式

所谓下标脚本语法就是能够通过, 实例[索引值]来访问实例中的数据

类似于以前我们访问数字和字典, 其实Swift中的数组和字典就是一个结构体

struct Student {
    var name:String = "lnj"
    var math:Double = 99.0
    var chinese:Double = 99.0
    var english:Double = 99.0

    func score(course:String) ->Double?
    {
        switch course{
            case "math":
                return math
            case "chinese":
                return chinese
            case "english":
                return english
            default:
                return nil
        }
    }

    // 要想实现下标访问, 必须实现subscript方法
    // 如果想要通过下标访问, 必须实现get方法
    // 如果想要通过下表赋值, 必须实现set方法
    subscript(course:String) ->Double?{
        get{
            switch course{
            case "math":
                return math
            case "chinese":
                return chinese
            case "english":
                return english
            default:
                return nil
            }
        }
        set{
            switch course{
            case "math":
                // 因为返回的是可选类型
                math = newValue!
            case "chinese":
                chinese = newValue!
            case "english":
                english = newValue!
            default:
                print("not found")
            }

        }
    }
}

类的定义：
Swift的类的定义与结构体相似，但又有所不同。

类是具有相同的属性和方法的抽象。

格式：
class 类名 {

}

表达式运算符：

===：恒等表达式，判断是否是同一个实例，也就是指向同一块存储空间。

!==：

属性

  1. 存储属性:其实Swift中的存储属性就是以前学习OC中的普通属性, 在结构体或者类中定义的属性, 默认就是存储属性
  2. 常量存储属性:只能在定义时或构造时修改, 构造好一个对象之后不能对常量存储属性进行修改
  3. 结构体和枚举都是值类型。
  4. 类是引用类型。

eg:

struct Person3 {
    var name: String
    var age: Int
}

let p3: Person3 = Person3(name: "lnj", age: 30)
// 因为结构体是值类型, 所以不能修改结构体常量中的属性
// 不能修改结构体/枚举常量对象中的值, 因为他指向的对象是一个常量
//p3.name = "hm"

// 不能修改结构体常量对象的值
//p3 = Person(name: "hm", age: 50)

class Person4 {
    var name: String = "lnj"
    var age: Int = 30
}

let p4:Person4 = Person4()
// 可以修改类常量中的值, 因为他指向的对象不是一个常量
p4.name = "hm"
// 不可以修改类常量的指向
//p4 = Person4()

懒加载

延迟存储属性:Swift语言中所有的存储属性必须有初始值, 也就是当构造完一个对象后, 对象中所有的存储属性必须有初始值, 但是也有例外, 其中延迟存储属性可以将属性的初始化推迟到该属性第一次被调用的时候.

懒加载应用场景:

  1. 有可能不会用到
  2. 依赖于其它值

注意点:

  1. 如果不是lazy属性, 定义的时候对象还没有初始化, 所以不能访问self
  2. 如果加上lazy, 代表使用时才会加载, 也就是使用到属性时才会调用self
  3. 访问一个类的属性必须通过对象方法, 所以访问时对象已经初始化完成了, 可以使用self

eg:

//懒加载的使用。
class Person: NSObject {

    var start:Double = 0.0
    var end:Double = 0.0

    lazy var length:Double = self.getLenght()

    //懒加载的使用。
    lazy var child:Array<Any> = {
        print("懒加载")
        var array = []
        return array as [AnyObject]

    }()

    func getLenght() -> Double {
        print("懒加载")
        return self.end - self.start
    }

}

计算属性

  1. Swift中的计算属性不直接存储值，跟存储属性不同，没有任何”后端存储与之对应”。
  2. 计算属性用于计算，可以实现getter和setter这两种计算方法。
  3. 枚举不可以有存储属性，但允许有计算属性。

setter:对象.属性 = 值

getter:var value = 对象.属性

struct Rect {
    var size:(width:Double,heigh:Double) = (0,0)
    var origin:(x:Double,y:Double) = (0,0)

    var center:(centerX:Double,centerY:Double) {
        get{
            return (size.width*0.5+origin.x,size.heigh*0.5+origin.y)
        }
        set{
            //注意: 计算属性不具备存储功能, 所以不能给计算属性赋值, 如果赋值会发生运行时错误
            //注意: setter可以自己传递一个参数, 也可以使用系统默认的参数newValue
            //如果要使用系统自带的参数, 必须删除自定义参数
            origin.x = newValue.centerX-size.width*0.5
            origin.y = newValue.centerY-size.heigh*0.5
        }
    }
}

var frame = Rect(size: (width: 20, heigh: 40), origin: (x: 0, y: 10))

frame.center = (30,20)

frame.origin

属性观察器

属性观察器，类似OC中的KVO，可以监听属性什么时候被改变，只有属性发生改变才会调用。

  1. willSet,将要设置新值
  2. didSet,设置完新值

注意点:

可以为除了计算属性和懒加载属性之外的存储属性添加属性存储器，但是可以在子类中为父类的计算属性添加属性观察器。因为在计算属性中也可以监听到属性的改变, 所以给计算属性添加属性观察器没有任何意义。

class Line3 {
    var start:Double = 0.0{
        willSet{
            print("willSet newValue = \(newValue)")
        }
        didSet{
            print("didSet oldValue = \(oldValue)")
        }
    }
    var end: Double = 0.0
}
var l = Line3()
l.start = 10.0

类属性

在结构体和枚举中用static,在类中使用class，并且在类中不允许将存储属性设置为类属性。

类属性和普通属性的区别：

  1. 普通属性是每个对象一份。
  2. 类属性是所有对象公用一份。

eg:

struct Person5 {
    // 普通的属性是每个对象一份
    var name: String = "lnj"
    // 类属性是素有对象共用一份
    static var gender:String = "man"
    //只读存储属性
    static var age:Int{
        return 30
    }
    func show()
    {
        print("gender = \(Person5.gender) name = \(name)")
    }
}

class Person6 {
// 普通的属性是每个对象一份
var name: String = "lnj"
// 类中不允许将存储属性定义为类属性
//class var gender:String = "man"
    // 类中只能将计算属性定义为类属性
    class var age:Int{
        return 30
    }

    func show()
    {
        print("age = \(Person6.age)")
    }
}

方法

方法：每一个类或者结构体的函数

方法分为类方法和实例方法。实例方法属于对象通过对象来调用，实例方法属于类通过类来调用。

class OnePerson {
    var name:String = "2"
    var age:Int = 20

    // 如果不希望某个参数作为外部参数, 可以在参数前面加上_, 忽略外部参数
    func setInfo(_name:String,_age:Int) {
        name = _name
        age = _age
    }

    class func show() {
        print("1")
    }
}

self关键字

self关键字, Swift中的self和OC中的self基本一样. self指当前对象, 如果self在对象方法中代表当前对象. 但是在类方法中没有self

mutating方法

值类型(结构体和枚举)默认方法是不可以修改属性的, 如果需要修改属性, 需要在方法前加上mutating关键字, 让该方法变为一个改变方法

值类型(结构体和枚举)默认方法是不可以修改属性的, 如果需要修改属性, 需要在方法前加上mutating关键字, 让该方法变为一个改变方法

struct Person3 {
    var name:String = "lnj"
    var age:Int = 30
    // 值类型(结构体和枚举)默认方法是不可以修改属性的, 如果需要修改属性, 需要在方法前加上mutating关键字, 让该方法变为一个改变方法
    // 注意: 类不需要, 因为类的实例方法默认就可以修改
    mutating func setName(name:String, age:Int)
    {
        self.name = name
        self.age = age
    }
    func show()
    {
        print("name = \(name) age = \(age)")
    }
}

enum LightSwitch{

Swift结构体

结构体

结构体是用来封装类型相同或者不同的数据。Swift中的结构体是一类类型，可以定义属性和方法，甚至构造函数和析构方法。

格式：

stuct 结构体名称 {
    结构体属性和方法
}

eg:

struct Size {
    var width:Double = 0.0
    var heigh:Double = 0.0
}

// 如果结构体的属性有默认值, 可以直接使用()构造一个结构体
// 如果结构体的属性没有默认值, 必须使用逐一构造器实例化结构体
var size = Size(width: 20, heigh: 30)
size.width = 04

结构体构造器

  1. Swift中的构造体和类和其他面向对象语言一样都有构造器，但OC没有。
  2. Swift要求实例化一个结构体或类的时候,所有的成员变量都必须有初始值, 构造函数的意义就是用于初始化所有成员变量的, 而不是分配内存, 分配内存是系统帮我们做的.
  3. 如果结构体中的所有属性都有默认值, 可以调用()构造一个结构体实例
  4. 如果结构体中的属性没有默认值, 可以自定义构造器, 并在构造器中给所有的属性赋值
    其实结构体有一个默认的逐一构造器, 用于在初始化时给所有属性赋值

结构体的成员方法

在C和OC中的结构体只有属性，Swift中的结构体中可以定义方法。

struct Size {
    var width:Double = 0.0
    var heigh:Double = 0.0

    func getWith() -> Double {
        return width
    }

    func showInfo() -> Void {
        print("width: \(width) heigh: \(heigh)")
    }
}

枚举:

Swift中的枚举比OC中的枚举强大，因为Swift中的枚举是一等类型，它可以想类和结构体一样增加属性和方法。

格式：

enum EnumName {
    case 枚举值
}

enum operation {
    case Add,Sub,Mul,Div
}

//可以使用常亮或者变量来接受枚举值
let operation:Operation = .Add
// 注意: 如果变量或常量没有指定类型, 那么前面必须加上该值属于哪个枚举类型
let operation1 = Operation.Div

枚举与Swift结合
// 注意: 如果case中包含了所有的值, 可以不写default. 如果case中没有包含枚举中所有的值, 必须写default
switch operation {
    case Operation.Add:
        print("1")
    case Operation.Sub:
        print("1")
    case Operation.Mul:
        print("1")
    case Operation.Div:
        print("1")
}

枚举中的原始值:OC中枚举的本质就是整数,所以OC中的枚举是有原始值的,默认是从0开始而Swift中的枚举默认是没有原始值的, 但是可以在定义时告诉系统让枚举有原始值。

//和OC中的枚举一样, 也可以指定原始值, 后面的值默认+1
//Swift中的枚举除了可以指定整形以外还可以指定其它类型, 但是如果指定其它类型, 必须给所有枚举值赋值, 因为不能自动递增
enum Operation1:Int {
    case Add = 1 ,Sub,Mul,Div
}

// rawValue代表将枚举值转换为原始值, 注意老版本中转换为原始值的方法名叫toRaw
print(Operation1.Add.rawValue)

通过原始值创建枚举值

  1. 原始值区分大小写。
  2. 返回的值是一个可选类型。以为原始值对应的枚举值不一定存在。
  3. 老版本中为fromRaw(“add”)

eg:

let enumValue = Operation1(rawValue: "1")

枚举相关值:

可以让枚举对应的原始值不是唯一，而是一个变量。

enum lineSegmentDescriptor{
    case StartAndEndPattern(start:Double, end:Double)
    case StartAndLengthPattern(start: Double, length:Double)
}

var lsd = lineSegmentDescriptor.StartAndLengthPattern(start: 0.0, length: 100.0)
lsd = lineSegmentDescriptor.StartAndEndPattern(start: 0.0, end: 50.0)
// 提取关联值优化写法
switch lsd
{
        case let .StartAndEndPattern(s, e):
            print("start = \(s) end = \(e)")
        case .StartAndLengthPattern(let s, let l):
            print("start = \(s) lenght = \(l)")
}

闭包:

  1. 函数是闭包的一种特殊。
  2. 闭包表达式（匿名闭包）- 能捕获上下文中闭包的值。

闭包的语法:

  1. in关键字是便于区分返回值和执行语句。
  2. 闭包表达式的类型和函数的类型一样，是参数上加返回值。也就是in之前的部分。
  3. 如果闭包是最后一个参数, 可以直接将闭包写到参数列表后面, 这样可以提高阅读性. 称之为尾随闭包

eg:

//完整写法
let say:(String)->String = {
    (name:String) -> String in
    return "hello \(name)"
}

//没有返回值的写法
let say:(String)->Void = {
    (name:String) in
    print("hello \(name)")
}

//无参无返回值
let say = {
    print("hello")
}

func bubbleSort(inout array:[Int], cmp: (Int, Int) -> Int)
{
    let count = array.count;
    for var i = 1; i < count; i++
    {
        for var j = 0; j < (count - i); j++
        {
            if cmp(array[j], array[j + 1]) == -1
            {
                let temp = array[j]
                array[j] = array[j + 1]
                array[j + 1] = temp
            }
        }
    }
}

// 闭包作为参数传递
bubbleSort(&arr, cmp: {
    (a: Int, b: Int) -> Int in
    if a > b{
        return 1;
    }else if a < b
    {
        return -1;
    }else
    {
        return 0;
    }
})

闭包表达式的优化:

  1. 类型优化, 由于函数中已经声明了闭包参数的类型, 所以传入的实参可以不用写类型
  2. 返回值优化, 同理由于函数中已经声明了闭包的返回值类型,所以传入的实参可以不用写类型
  3. 参数优化, swift可以使用$索引的方式来访问闭包的参数, 默认从0开始
  4. 如果只有一条语句可以省略return

eg:

bubbleSort(&arr){
//    (a , b) -> Int in
//    (a , b) in
    if $0 > $1{
        return 1;
    }else if $0 < $1
    {
        return -1;
    }else
    {
        return 0;
    }
}

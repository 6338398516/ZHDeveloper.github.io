#命名

在项目开发中，命名的规范非常重要。一份规范的代码，能让阅读者审核代码的时候有阅读一首诗的感觉。每个程序员都是编程的艺术家，不同的程序员有不同的编程习惯、编程方式。但是作为一个团队，我们应该有自己的一套规范，以方便维护代码。

####一、类名
类名（以及协议名）应以首字母大写，并按照驼峰命名规则来定义。

//类名的定义
@interface MediaPlayerViewController : UIViewController

@end

//分类的定义
@protocol UITableViewDelegate<NSObject, UIScrollViewDelegate>

@end

####二、分类
分类的命名由基类和拓展类名组成。以NSData为基类，创建NSData的分类，那么需要创建NSData+Util.h和NSData+Util.m文件。

//NSData+Util.h声明分类
@interface NSData (Util)

@end

//NSData+Util.m实现分类
@implementation NSData (Util)    

@end

####三、变量名

变量名应该以小写字母开头，并使用驼峰格式。尽量为变量起一个描述性的名字。

@property (nonatomic,strong) MediaControl *mediaControl;

####四、常量
常量名应该以小写字母 k 开头，使用驼峰格式分隔单词，如：kInvalidHandle，kWritePerm。

####五、方法的命名
方法名应该以小写字母开头，并混合驼峰格式。每个具名参数也应该以小写字母开头。方法的命名，要做到可读性高，简洁清晰。

insertObject:atIndex:

####六、枚举

//常规的枚举定义    
typedef enum {

    MediaPlaerStatusUnknow,
    MediaPlaerStatusPlay,
    MediaPlaerStatusPause,
    MediaPlaerStatusSeeking,
    MediaPlaerStatusCaching,
    MediaPlaerStatusFinish

} MediaPlaerStatus;

//高逼格的定义
typedef enum {

    MediaPlaerStatusUnknow  = 0,
    MediaPlaerStatusPlay    = 1<<0,
    MediaPlaerStatusPause   = 1<<1,
    MediaPlaerStatusSeeking = 1<<2,
    MediaPlaerStatusCaching = 1<<3,
    MediaPlaerStatusFinish  = 1<<4

} MediaPlaerStatus;

//规范的定义
typedef NS_ENUM(NSUInteger,MediaPlaerStatus) {

    MediaPlaerStatusUnknow  = 0,
    MediaPlaerStatusPlay    = 1<<0,
    MediaPlaerStatusPause   = 1<<1,
    MediaPlaerStatusSeeking = 1<<2,
    MediaPlaerStatusCaching = 1<<3,
    MediaPlaerStatusFinish  = 1<<4

};

#注释

适当的注释能提高代码的可读性。有些方法名定义晦涩难懂，可以选择添加注释来对方法的功能已经参数进行说明。注释不仅是写给别人看的，即便是自己写的方法、类、实例变量，添加注释便能一目了然。

####一、文件注释
每个文件的开头以文件内容的简要描述起始，紧接着是作者，最后是版权声明和/或许可证样板。使用Xcode创建的文件，就已经带有这些基本信息。

//
//  ViewController.m
//  MYCS
//
//  Created by AdminZhiHua on 16/3/28.
//  Copyright © 2016年 AdminZhiHua. All rights reserved.
//

####二、声明注释
每个类、方法、代理方法都应该添加相应的注释，以描述其相应的功能。

推荐安装使用VVDocumenter插件，具体安装就不多说。以下为我的VVDocumenter插件的配置。

方法的注释：

/*!
 *  @author zhihua, 16-03-28 13:03:44
 *
 *  @brief 判断WebView是否能够返回上一个页面
 *
 *  @return BOOL
 */
- (BOOL)canGoBack;

类属性的注释：建议使用///来进行注释，只需要写一句注释，并且有提示。

///  当前网页的RUL
@property (nonatomic,copy) NSString *urlString;

方法分区注释：

//方法分区格式：
#pragma mark - <注释，也可不写，没有注释时就只显示一条分割线>
#pragma mark 注释

//建议在文件中添加必要的方法分区注释
#pragma mark - life cycle

#pragma mark - system class delegate

#pragma mark - custom delegate

#pragma mark - action event

#pragma mark - private method

#pragma mark - public method

#pragma mark - getter and setter

#代码格式化

#调试技巧

LLDB是Xcode内置的调试工具，LLDB常用的命令有哪些？

  • breakpoint 设置断点定位到某一个函数
  • n 断点指针下一步
  • po打印对象

更多 LLDB 调试命令可查看

  1. The LLDB Debugger
  2. 苹果官方文档： iOS Debugging Magic
  3. 浅谈LLDB调试器

以下介绍一下facebook开源的LLDB调试工具chisel

//在终端通过Homebrew来安装chisel
brew update
brew install chisel

//在用户根目录下新建.lldbinit,并添加
# ~/.lldbinit
...
command script import /usr/local/opt/chisel/libexec/fblldb.py

chisel工具支持自定调试命令，具体设置查看官方文档。

上图是chisel已经定义的命令，常用的命令有：

  • hide/show——隐藏view/显示view

  • visualize——使用预览工具来显示view

  • border/unborder-给view设置边框，已经边框色/取消边框设置

  • caflush——当修改view的外观之后，可以让界面立即刷新显示。

  • pvc——输出当前控制器的结构。

  • pinternals——对象的属性

  • pviews——打印当前视图的层级结构。

  • taplog——点击屏幕，打印触摸到的view。

  • presponder——打印响应者链条。
